Надежное хранение

Не решена

Хранить данные нужно надежно, а вот системы при этом не всегда надежны.

В команде обработки данных некоторой компании Х могли бы использовать следующий алгоритм:

  • изначально все данных хранятся на одном узле

  • на отдельной итерации распределения данных выполняются 2 шага:

    1. выбирается некоторое число kkk не менее 1, но меньше количества записей на каждом задействованном в данный момент узле

    2. записи данных на каждом узле делятся на две части, одна из которых содержит ровно kkk записей, и меньшая по количеству записей часть переносится на новый узел

Таким образом, после первой операции распределения данных будет задействовано 2 узла, а после трех --- 8.

На данный момент у сервиса доступно nnn узлов, причем на iii-м из них хранится aia_iai​ записей.

Определите, можно ли получить текущее состояние серверов процессом распределения данных без ошибок, т.е. без потери записей и отключения некоторых серверов.

За решение будут выставлены баллы только в том случае, если дан правильный ответ на более чем 50% тестов.

Формат ввода

В первой строке содержится число nnn --- количество серверов (1≤n≤1051 \le n \le 10^51≤n≤105).

Во второй строке содержатся разделенные пробелами числа aia_iai​ --- количество записей на iii-м сервер (1≤ai≤10181 \le a_i \le 10^{18}1≤ai​≤1018).

Обратите внимание, что числа помещаются в 64-битный тип, но не помещаются в 32-битный.

Формат вывода

В единственной строке выведите YES, если текущее состояние может быть получено распределением данных без ошибок, иначе выведите NO.

Пример 1

Ввод
4
1 2 6 1
Вывод
YES

Пример 2

Ввод
4
1 2 3 4
Вывод
NO
